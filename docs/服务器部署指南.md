## 第一步：连接服务器

### Windows 自带方式（推荐，无需装软件）

打开 **PowerShell**（右键开始菜单 → Windows PowerShell）：

```powershell
ssh root@8.219.229.218
```

首次连接会提示 `Are you sure you want to continue connecting?`，输入 `yes` 回车。

> 如果购买服务器时设置了 root 密码，输入密码即可登录。
> 如果用的是密钥登录，需要指定密钥文件路径，见下方补充说明。

### 补充：密钥登录

如果你购买时创建了密钥（.pem 文件），把密钥放到 `C:\Users\你的用户名\.ssh\` 目录下，然后：

```powershell
ssh -i C:\Users\你的用户名\.ssh\密钥文件名.pem root@8.219.229.218
```

> 密钥文件权限要设为仅自己可读。如果 SSH 报错 `Permissions are too open`，在 PowerShell 中执行：
> ```powershell
> icacls "C:\Users\你的用户名\.ssh\密钥文件名.pem" /inheritance:r /grant:r "$env:USERNAME:R"
> ```

## 第二步：本地打包 + 上传文件

### 2.1 本地打包（在 IDEA 终端或 PowerShell 中执行）

```powershell
cd E:\ideaproject\consultant3
mvn clean package -DskipTests
```

看到 `BUILD SUCCESS` 说明打包完成。确认 JAR 存在：

```powershell
dir target\consultant3-0.0.1-SNAPSHOT.jar
```

### 2.2 在服务器上创建目录

先 SSH 登录服务器，然后：

```bash
mkdir -p /opt/consultant/content
```

### 2.3 上传文件（在本地 PowerShell 中执行）

**不要在 SSH 里执行这些命令**，新开一个 PowerShell 窗口，在项目目录下逐条执行：

```powershell
scp target\consultant3-0.0.1-SNAPSHOT.jar root@8.219.229.218:/opt/consultant/

scp Dockerfile root@8.219.229.218:/opt/consultant/

scp docker-compose.deploy.yml root@8.219.229.218:/opt/consultant/

scp .env root@8.219.229.218:/opt/consultant/

scp src\main\resources\db\schema.sql root@8.219.229.218:/opt/consultant/

scp -r src\main\resources\content\* root@8.219.229.218:/opt/consultant/content/
```

> 如果 content 目录是空的（没有法律文档），最后一条不用执行。但部署后 RAG 检索功能会没有数据。

### 2.4 确认文件上传完整

回到 SSH 窗口，检查：

```bash
ls -la /opt/consultant/
```

应该看到这 5 个文件 + 1 个目录：

```
Dockerfile
docker-compose.deploy.yml
.env
schema.sql
consultant3-0.0.1-SNAPSHOT.jar
content/          ← 目录，里面是法律文档
```

## 第三步：配置防火墙

### 3.1 阿里云控制台操作

1. 打开阿里云控制台 → **轻量应用服务器** → 点击 `Docker-mjdq`
2. 左侧菜单点击 **防火墙**
3. 点击 **添加规则**：

| 应用类型 | 协议 | 端口 | 授权对象 |
|----------|------|------|----------|
| 自定义 | TCP | 8080 | 0.0.0.0/0 |

> 已有的 SSH 22 端口不要动，那是远程连接用的。
> **不要添加 3306、6379**——数据库端口暴露公网有安全风险。

### 3.2 防火墙不生效检查（可选）

如果添加规则后还是访问不了，SSH 进服务器关闭系统防火墙：

```bash
systemctl stop firewalld    # 关闭防火墙（轻量服务器通常已关）
systemctl status firewalld  # 确认状态是 inactive
```

## 第四步：启动服务

在 SSH 连接的服务器上执行：

```bash
cd /opt/consultant
docker compose -f docker-compose.deploy.yml up -d
```

首次启动会下载三个镜像（总共约 1.5GB），需要等 3~5 分钟。你会看到类似这样的输出：

```
[+] Pulling mysql (mysql:8.0)...
[+] Pulling redis (redis/redis-stack-server:7.4.0-v1)...
[+] Building app...
[+] Running 3/3
 ✔ Container consultant-mysql  Started
 ✔ Container consultant-redis  Started
 ✔ Container consultant-app    Started
```

### 4.1 查看启动日志

```bash
docker compose -f docker-compose.deploy.yml logs -f app
```

看到这行说明启动成功：

```
Started Consultant3Application in X seconds
```

按 `Ctrl + C` 退出日志查看（容器不会停止）。

### 4.2 确认三个容器都在运行

```bash
docker compose -f docker-compose.deploy.yml ps
```

三个容器 STATUS 都是 `Up` 才对。输出大概像这样：

```
NAME                STATUS
consultant-mysql    Up (healthy)
consultant-redis    Up (healthy)
consultant-app      Up
```

## 常见问题

### Q1：启动报错 `no such file: schema.sql`

确认 `schema.sql` 已上传到 `/opt/consultant/` 目录。如果 MySQL 容器已经启动过一次，删掉重来：

```bash
docker compose -f docker-compose.deploy.yml down -v   # -v 会删除数据库数据
docker compose -f docker-compose.deploy.yml up -d
```

### Q2：应用启动后马上退出，日志显示数据库连接失败

这是 MySQL 还没准备好。等 10 秒再执行：

```bash
docker compose -f docker-compose.deploy.yml restart app
```

### Q3：页面能打开但发送消息没反应

检查 DashScope API Key 是否有效。查看日志：

```bash
docker compose -f docker-compose.deploy.yml logs app | grep -i error
```

### Q4：内存不够（4GB 跑三个容器比较紧张）

给 MySQL 和 Redis 加内存限制。编辑服务器上的 `docker-compose.deploy.yml`，在每个 service 下加：

```yaml
    deploy:
      resources:
        limits:
          memory: 512M
```

或者在服务器上用 `vim` 改，或者本地改好重新上传 docker-compose.deploy.yml，然后重启：

```bash
docker compose -f docker-compose.deploy.yml up -d --force-recreate
```

### Q5：想更新代码重新部署

```powershell
# 本地重新打包
mvn clean package -DskipTests

# 上传新的 JAR
scp target\consultant3-0.0.1-SNAPSHOT.jar root@8.219.229.218:/opt/consultant/

# SSH 到服务器，重新构建并启动
ssh root@8.219.229.218 "cd /opt/consultant && docker compose -f docker-compose.deploy.yml up -d --build app"
```

## 附：服务器 Docker 常用命令

```bash
# 查看所有容器状态
docker ps -a

# 查看某个容器日志（最近 100 行）
docker logs --tail 100 consultant-app

# 重启某个容器
docker restart consultant-app

# 停止所有服务
docker compose -f docker-compose.deploy.yml down

# 停止并删除数据（慎用！会清空数据库）
docker compose -f docker-compose.deploy.yml down -v

# 进入容器内部排查
docker exec -it consultant-app /bin/bash
```
