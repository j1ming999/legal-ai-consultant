# 查询类型感知的动态混合检索设计

## 目标

在保留当前 RAG 检索链路所有能力的前提下，加入“查询类型感知的动态混合检索”用于测试，让混合检索在评测结果上优于纯向量检索。当前功能包括向量检索、关键词检索、加权 RRF 融合、去重、TopN 限制、查询改写与原始查询扩展检索，均继续保留。

## 方案选择

采用方案一：动态权重 + 关键词准入门槛。

不直接提高关键词权重，因为关键词 Top20 中可能包含大量噪声；也不做过度定向的评测集硬编码。改为先判断查询类型，再决定关键词结果是否有资格进入 RRF，并按查询类型设置关键词权重。

## 查询类型

在 `HybridContentRetriever` 内用规则判断查询类型，不额外调用模型：

1. 精确查询：包含“第X条”“第X款”“第X章”，或包含法规名称、书名号法律名称。
2. 法律术语查询：包含合同无效、工伤认定、遗产继承、劳动仲裁、交通事故、家庭暴力等核心法律术语。
3. 口语化查询：没有明显法条、法律名称、核心法律术语的普通表达。

## 动态权重

RRF 公式从固定权重变为每次查询动态选择权重：

```text
score = wv / (60 + rank_vector) + wk / (60 + rank_keyword)
```

初始测试权重：

| 查询类型 | 向量权重 | 关键词权重 |
| --- | ---: | ---: |
| 精确查询 | 1.0 | 1.5 |
| 法律术语查询 | 1.0 | 0.8 |
| 口语化查询 | 1.0 | 0.2 |
| 关键词结果质量差 | 1.0 | 0.0 |

构造器中的原有权重参数仍保留，作为默认/兼容配置；动态策略只在检索时根据查询覆盖实际融合权重。

## 关键词准入门槛

关键词检索仍可返回 TopN，但进入 RRF 前先过滤。关键词结果满足以下任一条件才参与融合：

1. 命中查询中的法律名称；
2. 命中查询中的法条编号；
3. 命中核心法律术语；
4. 命中至少 2 个有效关键词；
5. 关键词命中率达到阈值，初始为 0.4。

如果过滤后没有合格关键词结果，则关键词权重降为 0，仅返回向量融合结果，避免低质量关键词结果拉低 P@5。

## 测试设计

采用 TDD：先在 `HybridContentRetrieverTest` 中增加失败测试，再实现代码。

测试覆盖：

1. 精确法条查询提高关键词权重，关键词第一名能进入并靠前。
2. 口语化查询降低关键词权重，低质量关键词结果不会挤掉向量前排结果。
3. 关键词结果不满足准入门槛时不参与融合。
4. 满足核心法律术语或多个有效关键词时允许参与融合。
5. 现有 RRF 去重、maxResults、空输入测试继续通过。

## 验证

先运行 `HybridContentRetrieverTest`，再运行现有检索评测脚本或测试集。如果混合检索仍未超过向量检索，下一步只调查询类型规则、核心术语词表和阈值，不改变主结构。
